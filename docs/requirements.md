## 1. プロジェクト概要

### 1.1 背景・目的

個人投資家が保有する日本株ポートフォリオについて、過去の株価実績データをもとにモンテカルロシミュレーションを実施し、将来の評価額の確率分布を可視化するWebアプリケーションを開発する。

「このポートフォリオは10年後にどうなっているか」という問いに対して、単一の予測値ではなく確率分布として答えることで、リスクを定量的に把握した上での資産形成の意思決定を支援する。

### 1.2 対象ユーザー

- 日本株を保有する個人投資家
- ポートフォリオのリスク・リターン特性を把握したい方
- 長期投資の観点で将来の資産評価を確認したい方

### 1.3 提供価値

- 保有銘柄の過去株価データを自動取得し、統計量（平均リターン・標準偏差・相関）を算出
- モンテカルロシミュレーションによる将来評価額の確率分布の可視化
- 元本割れ確率・期待リターンなど直感的な指標の提示
- 銘柄ごとのリスク寄与度の把握
- 銘柄の追加・除外判断を支援するポートフォリオ分析機能の提供

## 2. 機能要件

### 2.1 銘柄入力機能

ユーザーが保有銘柄の情報を入力する。複数銘柄の登録・編集・削除が可能。

| 入力項目 | 形式 | 例 | 説明 |
|---------|------|-----|------|
| 証券コード | 4桁数字 | 7203 | 東証上場銘柄コード |
| 銘柄名 | 自動取得 | トヨタ自動車 | 証券コード入力後に自動補完 |
| 保有株数 | 整数 | 100 | 単元株未満も入力可 |
| 取得単価 | 円 | 2,800 | 1株あたりの購入価格 |

操作仕様：
- 証券コードを入力すると銘柄名を自動取得・表示
- 銘柄の追加・削除はダイナミックに行える（ページ遷移なし）
- 最大保有銘柄数：20銘柄
- 入力データはブラウザのローカルストレージに保存し、次回アクセス時に復元

### 2.2 株価データ取得機能

#### 2.2.1 データソース

| データソース | 状態 | 備考 |
|------------|------|------|
| Stooq | **現在使用中** | 無料・認証不要・日本株対応・前日終値まで取得可能。CSV形式で提供 |
| Yahoo Finance API（非公式） | 廃止 | レート制限強化（Cookie + Crumb 認証必須化）により安定利用が困難なため、Stooq に移行（2026年3月） |

> **データソース移行の経緯**
> 当初 Yahoo Finance API（v8 非公式エンドポイント）を使用していたが、Yahoo Finance 側のアクセス制限強化により、認証なしのリクエストが `429 Too Many Requests` で拒否されるようになった。Cookie + Crumb 認証フローの実装を試みたが、レート制限が厳しく安定運用が困難と判断し、認証不要で安定した Stooq に移行した。

将来的な J-Quants API（JPX公式）への移行を考慮し、データ取得レイヤーはインターフェースで抽象化して実装する（4.2参照）。商用化の際は J-Quants API への移行を検討する。

#### 2.2.2 取得データ仕様

- 取得期間：直近5年間（デフォルト）／3年・10年を選択可能
- 取得頻度：日次終値
- データ項目：日付・始値・高値・安値・終値・出来高
- 終値を使用（Stooq の終値は株式分割調整済み。ただし配当落ち調整は含まれないため、配当分のリターンはわずかに過小評価される可能性がある）

#### 2.2.3 データ取得のタイミング

- 銘柄登録時に自動取得
- 「データ更新」ボタンで手動再取得可能
- 取得済みデータはセッション中キャッシュし、不要なAPIリクエストを抑制

### 2.3 統計量計算機能

取得した株価データから以下の統計量を算出する。

**個別銘柄ごと**
- 日次リターン（対数リターン）の時系列
- 年率平均リターン（μ）
- 年率標準偏差（σ）：ボラティリティ
- シャープレシオ（無リスク金利：0.1%想定）
- 最大ドローダウン

**ポートフォリオ全体**
- 銘柄間の相関係数行列
- ポートフォリオの年率期待リターン（加重平均）
- ポートフォリオの年率標準偏差（共分散行列を考慮）
- 各銘柄のリスク寄与度（%）

### 2.4 シミュレーション計算機能

#### 2.4.1 計算方式

Phase 1（シンプル版）とPhase 2（本格版）の2段階で実装する。

**Phase 1：各銘柄独立モデル**
- 銘柄間の相関を無視し、各銘柄を独立したGBM（幾何ブラウン運動）でシミュレート
- 計算が単純で実装・検証が容易

**Phase 2：相関考慮モデル**
- 共分散行列のコレスキー分解を使用
- 銘柄間の相関を反映した多変量正規分布から乱数を生成
- より現実的なシミュレーション

#### 2.4.2 シミュレーションパラメータ

| パラメータ | 単位 | デフォルト値 | 説明 |
|-----------|------|------------|------|
| シミュレーション期間 | 年 | 10 | 1〜30年で選択可 |
| シミュレーション回数 | 回 | 10,000 | モンテカルロ試行回数 |
| 年次追加投資額 | 万円 | 0 | 毎年ポートフォリオに追加投資する金額 |
| 無リスク金利 | % | 0.1 | シャープレシオ計算用 |

#### 2.4.3 計算処理

- Web Worker を使用しUIをブロックしない
- 進捗バーで計算進捗を表示
- 計算時間の目安：10,000回 × 20銘柄 × 10年 を5秒以内に完了

### 2.5 結果表示機能

#### 2.5.1 ポートフォリオサマリー

現在の状況を一覧表示する。

| 指標 | 説明 |
|------|------|
| 現在の評価額合計 | 前日終値ベースの時価総額 |
| 取得価格合計 | 入力した取得単価 × 株数の合計 |
| 含み損益 | 評価額 − 取得価格合計 |
| 含み損益率 | % |
| ポートフォリオ期待リターン | 年率 % |
| ポートフォリオボラティリティ | 年率 % |
| シャープレシオ | — |
| 過去最大ドローダウン | 取得期間中の最大下落率（%）|

> **リスク許容度について**
> 本アプリではリスク許容度の入力は求めない。代わりに過去最大ドローダウン・悲観シナリオ（10パーセンタイル）・元本割れ確率などの情報を提示し、ユーザー自身が「このリスクに耐えられるか」を判断できる材料を提供する。

#### 2.5.2 シミュレーション結果サマリーカード

| 指標 | 説明 |
|------|------|
| N年後 期待評価額（中央値） | 万円 |
| N年後 楽観シナリオ（90パーセンタイル） | 万円 |
| N年後 悲観シナリオ（10パーセンタイル） | 万円 |
| 元本割れ確率 | 取得価格を下回る確率（%） |
| 2倍達成確率 | 取得価格の2倍を超える確率（%） |

#### 2.5.3 グラフ表示

**グラフ①：将来評価額の確率分布（ヒストグラム）**
- N年後の評価額の度数分布を表示
- 10・50・90パーセンタイルのラインを表示
- 取得価格総額のラインを表示（元本割れの境界）

**グラフ②：評価額の年次推移（帯グラフ）**
- X軸：経過年数（0〜N年）
- Y軸：ポートフォリオ評価額（万円）
- 10〜90パーセンタイルの帯と中央値ラインを表示
- 取得価格総額のラインを重ねて表示

**グラフ③：元本割れ確率の推移**
- X軸：経過年数
- Y軸：元本割れ確率（%）
- リスクが時間とともにどう変化するかを可視化

**グラフ④：銘柄別リスク寄与度・リターン寄与度（並列円グラフ）**
- 各銘柄がポートフォリオ全体のリスク・リターンに占める割合をそれぞれ表示
- 2つの円グラフを並列配置し、リスクとリターンのバランスを視覚的に比較
- 銘柄の集中リスクの把握に使用

**グラフ⑤：銘柄別過去株価の推移（折れ線グラフ）**
- 取得期間中の各銘柄の株価推移
- 正規化表示（取得時点を100として指数化）で銘柄間の比較を容易にする

**グラフ⑥：銘柄間相関行列（ヒートマップ）**
- 保有銘柄間の相関係数を色付きのマトリックスで表示
- 分散投資の効果を視覚的に確認

**グラフ⑦：限界リスク寄与（棒グラフ）**
- 各銘柄のウェイト変動に対するポートフォリオリスクの感度を表示
- リスク低減に寄与している銘柄とリスクを押し上げている銘柄を識別

#### 2.5.4 ポートフォリオ分析機能

銘柄の追加・除外の意思決定を支援するための分析機能を提供する。

**（A）銘柄間相関行列**

保有銘柄間の相関係数をヒートマップ形式で可視化する。

- 相関係数行列は統計量計算機能（2.3）で算出済みのものを表示
- セルの色で相関の強弱を直感的に把握（強い正の相関：赤系、無相関：白、強い負の相関：青系）
- 対角線上にはボラティリティを表示
- 分散投資の効果を確認し、相関の高い銘柄の偏りを発見するのに使用

**（B）銘柄別リターン寄与度**

各銘柄がポートフォリオ全体の期待リターンにどれだけ寄与しているかを表示する。

- 計算式：銘柄iのリターン寄与度 = wi × μi / Σ(wj × μj)
  - wi：銘柄iの評価額ウェイト、μi：銘柄iの年率期待リターン
- リスク寄与度（グラフ④）と並べて表示し、リスク・リターンのバランスを比較可能にする
- リスク寄与度に対してリターン寄与度が低い銘柄は、ポートフォリオ効率を下げている可能性がある

**（C）限界リスク寄与（Marginal Contribution to Risk）**

各銘柄を1単位（1%相当）増減した場合のポートフォリオリスクの変化を表示する。

- 計算式：MCRi = ∂σp/∂wi = (Σw)i / σp
  - Σ：共分散行列、w：ウェイトベクトル、σp：ポートフォリオのボラティリティ
- MCRが高い銘柄はウェイト増加時にリスクを大きく押し上げる
- MCRが低い（または負の）銘柄はポートフォリオのリスク低減に寄与している
- 棒グラフで表示し、各銘柄のリスクへの感度を一目で比較可能にする

**（D）除外シミュレーション（Leave-One-Out分析）**

各銘柄を1つずつ除外した場合のポートフォリオ指標を比較テーブルで表示する。

- 各銘柄について「その銘柄を除外した場合」のポートフォリオを構成し、以下を算出：
  - 期待リターン（年率）
  - ボラティリティ（年率）
  - シャープレシオ
  - 元本割れ確率（現在のシミュレーション期間で）
- 現在のポートフォリオの値を基準行として表示し、各除外パターンとの差分を色分け
  - シャープレシオが改善する場合：緑（除外すべき候補）
  - シャープレシオが悪化する場合：赤（保持すべき銘柄）
- 除外時のウェイトは残りの銘柄の評価額で按分（プロラタ方式）

> **運用上の注意**
> Leave-One-Out分析は過去データに基づく統計的な指標であり、将来の最適構成を保証するものではない。あくまで現在のポートフォリオ構成を見直す際の参考情報として活用すること。

## 3. 非機能要件

### 3.1 パフォーマンス

| 項目 | 要件 |
|------|------|
| 株価データ取得 | 1銘柄あたり3秒以内 |
| シミュレーション実行 | 10,000回 × 20銘柄 × 10年を5秒以内 |
| グラフ描画 | 計算完了後1秒以内 |
| 初期表示 | 3秒以内（LCP） |

### 3.2 対応環境

| 項目 | 対応範囲 |
|------|---------|
| ブラウザ | Chrome / Firefox / Safari / Edge（最新2バージョン） |
| デバイス | PC・タブレット・スマートフォン（レスポンシブ対応） |
| 画面解像度 | 375px〜1920px |

### 3.3 ユーザビリティ

- 証券コード入力時の銘柄名オートコンプリート
- 各指標へのツールチップ説明
- ポートフォリオデータのJSON形式でのエクスポート・インポート
- シミュレーション結果のPNG/PDFエクスポート
- データ取得エラー時のわかりやすいエラーメッセージ表示

### 3.4 データの取り扱い

- 入力データはブラウザのローカルストレージのみに保存（サーバー送信なし）
- 株価データのキャッシュはセッションストレージに保存
- 個人情報・証券口座情報は一切取り扱わない

### 4.2 データソース切り替えへの備え

Yahoo Finance API から J-Quants API への移行を見越し、データ取得レイヤーをインターフェースで抽象化する。プロバイダーを切り替えるだけでデータソースを変更できる設計にしておく。

```typescript
interface StockDataProvider {
  fetchHistoricalPrices(code: string, from: Date, to: Date): Promise<PriceData[]>
  fetchStockInfo(code: string): Promise<StockInfo>
}

class YahooFinanceProvider implements StockDataProvider { ... }  // 現在使用（実体は Stooq）
class JQuantsProvider implements StockDataProvider { ... }       // 商用化時に切り替え
```

> **注記**: クラス名は `YahooFinanceProvider` のまま残存しているが、実際のデータ取得先は Stooq に移行済み。今後リファクタリングの際に `StooqProvider` にリネーム予定。

**各プロバイダーの比較**

| 項目 | Stooq（現在使用） | Yahoo Finance API | J-Quants API |
|------|------------------|-----------------|--------------|
| 費用 | 無料 | 無料 | 無料〜16,500円/月 |
| 公式サポート | なし | なし（非公式） | あり（JPX公式） |
| 認証 | 不要 | Cookie + Crumb 必須 | APIキー |
| 直近データ | 前日終値まで | 前日終値まで | 無料プランは12週間遅延 / 有料プランはリアルタイム |
| 調整後終値 | 分割調整のみ（配当調整なし） | 分割・配当調整済み | 分割・配当調整済み |
| データ形式 | CSV | JSON | JSON |
| 利用規約 | 非商用のみ | グレーゾーン（非商用のみ推奨） | 明確（商用利用可） |
| 安定性 | 高い | 低い（レート制限強化） | 高い |

## 7. 考慮事項・制約事項

### 7.1 計算上の簡略化

- 配当金・株主優待は評価額に含めない（株価の値動きのみを対象）
- 税金（譲渡所得税・配当税）は計算に含めない
- 売買手数料は考慮しない
- 上場廃止リスクは考慮しない
- 過去の統計量が将来も継続すると仮定（定常性の仮定）

### 7.2 データソースに関するリスク

Stooq は公式サポートのない無料サービスであり、仕様変更・サービス停止により予告なく使用不可になる可能性がある。個人・非商用利用の範囲で使用する。商用化の際は J-Quants API（JPX公式）への移行を検討する。

> **Yahoo Finance API からの移行について**
> 当初使用していた Yahoo Finance API（非公式 v8 エンドポイント）は、2026年3月時点で Cookie + Crumb 認証が必須化され、認証なしのリクエストは `429 Too Many Requests` で拒否されるようになった。レート制限も厳しく、短時間の連続アクセスで IP 単位のブロックが発生する。これにより安定利用が困難と判断し、Stooq に移行した。なお、Stooq は配当落ち調整を含まないため、配当分のリターンがわずかに過小評価される点に留意が必要。

### 7.3 免責事項

本ツールの計算結果はあくまで参考値であり、将来の株価・運用成果を保証するものではありません。投資判断はご自身の責任で行い、必要に応じて専門家への相談を推奨します。
